# lecture-ux-review — 강의 도구 UX 검수 가이드

이 도구는 **강사가 청중 앞에서 실시간으로** 쓰는 프레젠테이션 도구다. 데스크톱
앱처럼 천천히 탐색하는 것이 아니라, 강의 흐름 속에서 **순간적으로** 쓰인다.
모든 검수는 "강의 중 한 손으로, 화면을 보면서, 1~3초 안에" 기준으로 한다.

## 1. 라이브 강의 핵심 휴리스틱 (왜 중요한가)

| 원칙 | 강의 현장 이유 | 점검 질문 |
|------|---------------|-----------|
| **속도** | 강사는 말하면서 도구를 바꾼다 | 도구 전환이 클릭/키 한 번에? 단축키가 직관적인가? |
| **가시성** | 뒷자리·밝은 강의실·빔프로젝터 | 색·선 두께·커서·토스트가 큰 화면에서 또렷한가? |
| **방해 최소화** | 청중 화면에 잡동사니 노출 금지 | present 중 UI가 숨는가? 토스트가 발표를 가리지 않나? |
| **회복 가능성** | 실수해도 강의가 안 끊겨야 | 실행취소가 있는가? 잘못 지워도 복구되나? |
| **예측 가능성** | 강사가 손가락 기억으로 쓴다 | 같은 키/제스처가 항상 같은 결과? PowerPoint 관습 따르나? |
| **무마크 지시** | 가리키되 더럽히지 않기 | 레이저/스포트라이트가 있는가? |

## 2. 검수 체크리스트

**도구·인터랙션**
- [ ] 활성 도구가 시각적으로 명확히 구분되는가(active 상태)
- [ ] 커서가 현재 도구를 알려주는가(펜=crosshair, 텍스트=text, 지우개=원형 등)
- [ ] 단축키가 발견 가능한가(툴팁에 키 표기)
- [ ] 자주 쓰는 동작(도구 전환·페이지 이동·실행취소)이 손에 가까운가

**가시성**
- [ ] 기본 색 4종이 빔프로젝터에서 구분되는가(특히 노랑은 흰 배경에서 약함)
- [ ] 선 두께 기본값이 큰 화면에서 충분히 굵은가
- [ ] 토스트/모달 텍스트 대비가 충분한가(라이트/다크 모두)

**전체보기(present) 모드**
- [ ] 발표 중 불필요한 chrome(푸터·힌트)이 숨는가
- [ ] 강사 전용 정보(타이머 등)는 보이되 청중 방해 최소인가
- [ ] Esc/단축키 안내가 과하게 노출되지 않는가

**오류·피드백**
- [ ] 잘못된 파일·실패 시 명확한 토스트가 뜨는가
- [ ] 위험 동작(전체 삭제, PDF 교체)에 대한 피드백이 적절한가

## 3. 흔한 안티패턴
- **메뉴 사냥**: 자주 쓰는 기능이 깊은 메뉴/모달 안에 → 강의가 끊김
- **약한 가시성**: 얇은 선·연한 색이 뒷자리에서 안 보임
- **소리 없는 실패**: 동작이 안 됐는데 피드백이 없음
- **present 누수**: 발표 모드에서 개발/디버그 UI나 푸터가 청중에게 노출
- **관습 위반**: 화살표=페이지인데 어떤 화면선 다르게 동작 → 예측 깨짐
- **복구 불가**: 실행취소 없이 실수가 영구화

## 4. 발견 우선순위
- **치명적(P0)**: 강의를 중단시킴(앱 멈춤, 주석 유실, 발표 중 누수)
- **중요(P1)**: 흐름을 방해(느린 전환, 안 보이는 색, 피드백 부재)
- **사소(P2)**: polish(정렬·간격·미세 대비)

## 5. 리포트 형식
`_workspace/NN_lecture-ux-reviewer_review.md`에 우선순위별로:
- **현상** — 무엇이 문제인가
- **강의 현장 영향** — 실제 강의에서 어떤 지장이 생기는가
- **개선안** — 구체적 수정(가능하면 CSS 값·동작 제안)
- 가능하면 스크린샷 첨부(라이트/다크, 일반/present 비교)

주관적 취향은 "권장"으로 표기하고 강제하지 않는다. 검증이 필요한 가설은
canvas-qa에 실측을 요청한다.
